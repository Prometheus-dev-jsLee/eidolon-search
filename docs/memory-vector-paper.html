인간 기억 구조의 벡터DB 모사 가능성에 관한 사고실험

의식/무의식 이중 레이어 아키텍처와 멀티모달 기억 벡터 설계를 중심으로

유형 · 사고실험 기반 설계 제안
작성 · 2025
초록 Abstract

본 논문은 인간 기억의 신경과학적 구조를 벡터 데이터베이스(Vector DB) 패러다임으로 모사하는 가능성을 탐색한다. 기존 접근의 한계—벡터DB에 기억 전체를 투영하려는 시도—를 지적하고, 의식(conscious)과 무의식(unconscious)의 역할을 명확히 분리하는 이중 레이어 아키텍처를 제안한다. 의식을 세션 컨텍스트 기반의 I/O 처리 레이어로, 무의식을 저장·압축·재인덱싱을 담당하는 백그라운드 레이어로 위치시키고, 순수 수치 벡터 대신 텍스트·음성·이미지·영상을 복합적으로 담은 멀티모달 페이로드 구조를 제안한다. 이를 통해 기억 왜곡, 추억 미화, 감정적 기억 강화 등의 인간적 기억 현상이 시스템적으로 재현 가능함을 논증한다.

Section 01

감각 정보의 차원 분해: 오감은 몇 차원인가

벡터DB에서 "차원(dimension)"이란 하나의 개념을 설명하기 위해 필요한 독립적인 특징(feature)의 수다. 예를 들어 [달콤함, 빨간색, 구형, 과일향]은 사과를 4차원으로 표현한 것이다. 핵심 조건은 각 축이 서로 독립적인 정보여야 한다는 것이다.

이 조건 하에 인간의 오감을 차원으로 분해하면 다음과 같다.

👁️ 시각 — 최소 수백만 차원

가장 고차원적 감각으로, 색상(가시광선 주파수), 공간 위치(x, y, z, 심도), 움직임 방향과 속도, 형태(엣지·곡률·텍스처·대칭성) 등 다수의 독립 축이 존재한다. 인간이 구별 가능한 색상만 약 1,000만 가지에 달하며, 현대 이미지 임베딩(CLIP 등)이 1,024~4,096차원을 사용함에도 이는 대폭 압축된 결과다.

👂 청각 — 수천~수만 차원

인간이 구별하는 음정만 약 1,400개이며, 여기에 음량, 음색(배음 구조), 시간적 패턴(리듬), 공간적 위치(좌우/거리) 등이 추가된다. 음성 언어 정보가 포함되면 의미론적 차원이 폭발적으로 증가한다.

👃 후각 — 약 400차원

인간의 후각 수용체는 약 400종이며, 그 조합으로 이론상 약 1조 가지 냄새를 구별할 수 있다. 후각은 변연계(감정·기억 중추)에 직접 연결된 유일한 감각으로, 냄새 기억이 유독 강렬하고 오래 지속되는 생물학적 기반이다.

👅 미각 — 약 5~30차원

단맛·짠맛·신맛·쓴맛·감칠맛(umami)의 기본 5가지에 온도, 통증(매운맛), 탄산 자극을 포함하면 20~30차원에 해당한다. 단독으로는 저차원이지만, 후각과 결합해 "풍미(flavor)"를 형성하여 고차원화한다.

🖐️ 촉각 및 고유감각 — 수천 차원

압력·진동·온도·통증·간지러움은 각각 별개의 수용체를 가지며, 신체 부위별 공간 정보가 추가된다. 특히 고유감각(proprioception)—수백 개 관절 각도를 포함하는 신체 자세 정보—은 흔히 과소평가되지만 상당한 차원을 차지한다.

감각 외에 기억이 추가하는 차원들은 아래 표와 같다.

Section 02

기억 압축의 3단계 구조와 하한선 추정

인간의 감각 정보 처리는 의식에 진입하기 전부터 이미 극적인 압축이 이루어진다. 시신경의 신호량은 초당 약 10Mbps이나, 의식에 실제로 올라오는 정보는 약 50bps에 불과하다. 즉 감각기관 → 의식 단계에서만 약 20만 배 압축이 발생한다.

1단계: 작업기억(Working Memory) — 의식의 버퍼

밀러의 법칙(Miller's Law)에 따르면 인간이 한 번에 의식적으로 처리하는 단위는 약 7±2 청크(chunk)다. 청크 하나를 수십~수백 차원짜리 벡터로 가정할 때, 작업기억 전체의 표현 용량은 약 수백~수천 차원에 해당한다.

2단계: 해마의 인코딩 — 에피소드 압축

해마는 경험의 감각 데이터 전체가 아니라 "어떤 피질 패턴들을 재활성화할지에 대한 포인터(index)"를 저장하는 것에 가깝다. 해마의 장소세포(place cells)와 시간세포(time cells)가 시공간 좌표계를 형성하고, 편도체의 감정 태그와 피질의 의미 태그가 결합된다. 이 단계의 압축 차원 추정치는 약 100~300차원이다.

3단계: 장기기억 공고화 — 의미망 합류

반복되거나 감정적으로 강한 기억은 수면 중 느린파수면(SWS, Slow-Wave Sleep) 동안 재활성화되어 피질로 전이된다. 이 과정에서 에피소드의 구체적 감각 정보는 손실되고 패턴과 의미만 남는다. 오래된 기억의 추정 차원은 약 10~50차원이며, 수십 년이 지난 기억은 사실상 5~10차원의 감정적 요약으로 수렴한다.

Section 03

기억 왜곡과 추억 미화의 메커니즘

기억 압축과 재구성 구조는 필연적으로 기억 왜곡을 내포한다. 이는 시스템의 결함이 아니라, 압축-재구성 메커니즘의 구조적 부산물이다. 

# 기억의 생애 주기
원본 경험 (수십만 차원)
        ↓  [해마가 인코딩]
압축된 기억 벡터 (저차원)
        ↓  [회상 시 재구성]
"기억했다고 느끼는 것" (다시 수십만 차원으로 펼침)
        ↑
        빈 차원을 현재의 감정·기대·사후 정보로 채워 넣음

추억 미화(Rosy Retrospection)는 감정 차원(valence)의 값이 회상 시마다 현재 기분 상태에 의해 덮어써지는 현상으로 이해할 수 있다. 원본 감각 데이터는 변하지 않지만, 그것을 해석하는 감정 가중치가 교체된다.

목격자 기억 오염은 언어·개념 차원이 사후에 주입된 정보로 재인코딩되는 현상이다. 이는 벡터 공간에서 기존 벡터의 일부 축 값이 새로운 정보에 의해 갱신되는 것과 동일한 메커니즘이다.

냄새 하나가 20년 전 기억을 통째로 불러오는 것은, 그 후각 벡터(약 5~10차원)가 해당 기억 벡터와 코사인 유사도(cosine similarity)가 특히 높은 차원 축들을 공유하기 때문이다. 즉, 저차원의 감각 앵커가 고차원 기억 재구성의 트리거로 작동한다.

— 프루스트 현상(Proust Phenomenon)의 계산론적 해석

이 구조에서 결정적 통찰은 벡터가 기억 그 자체가 아니라 기억을 찾아가는 열쇠(key)라는 점이다. LLM이 벡터로 "어떤 기억 근처인지"를 탐색하고, 멀티모달 페이로드를 통해 그 기억을 의식 레이어에서 재구성한다. 이 분리가 재구성 오류(기억 왜곡)와 창의적 재해석을 동시에 가능하게 하는 구조적 기반이다.

Section 05

멀티모달 기억 벡터 설계

순수한 수치 벡터만으로는 인간 기억의 다차원성을 균등하게 커버하지 못한다. 현재의 384차원 텍스트 임베딩은 언어 의미 공간에 최적화되어 있으며, 시간적 위치, 신체 감각, 현저성(salience) 등의 축은 거의 인코딩되지 않는다.

제안하는 설계는 벡터를 인덱스 키로, 멀티모달 원본을 실제 내용으로 분리하는 구조다. 

class HumanMemoryRecord:
    # ── 인덱스 레이어 (무의식: 검색 및 유사도 계산용) ──
    vector: np.array        # 384차원 — 의미/감각 압축 벡터

    # ── 원본 페이로드 (무의식: 재구성 재료) ──
    text:  str              # 언어화된 서술 / 내부 독백
    audio: str | None       # 그때 들리던 소리 또는 음성 (파일 경로)
    image: str | None       # 시각적 앵커 이미지
    video: str | None       # 에피소드 영상 클립

    # ── 메타데이터 (무의식: 기억의 동적 속성) ──
    timestamp:       float  # 경험 발생 시각 (에피소드 좌표)
    valence:         float  # 감정가: -1.0(부정) ~ +1.0(긍정)
    arousal:         float  # 각성도: 0.0(잔잔) ~ 1.0(강렬)
    decay_rate:      float  # 망각 속도 (에빙하우스 곡선 파라미터)
    consolidation:   int    # 재활성화 횟수 (강화 정도)
    source_modality: list   # 인코딩 출처: ["visual","auditory","olfactory",...]
    social_context:  list   # 함께한 주체들의 임베딩 참조

검색 함수의 재설계

일반 벡터DB는 코사인 유사도만으로 검색하지만, 인간 기억 모사 시스템은 감정, 시간, 강화 정도를 검색 스코어에 통합해야 한다. 

def human_memory_search(query_vector, current_state):
    # 기본 의미 유사도
    semantic_score = cosine_similarity(query_vector, memory.vector)

    # 감정 친화도: 현재 기분과 비슷한 감정 기억이 더 잘 떠오른다
    emotion_score = 1 - abs(current_state.valence - memory.valence)

    # 시간 가중치: 최근 기억 우선, 단 강화된 기억은 예외
    time_weight = recency_weight(memory.timestamp)

    # 강화 가중치: 자주 회상된 기억은 더 쉽게 떠오른다
    recall_boost = log(1 + memory.consolidation) * 0.1

    # 현재 decay 상태 (잊혀가는 기억)
    current_strength = memory.decay_rate ** days_elapsed(memory.timestamp)

    return (
        semantic_score * 0.5
      + emotion_score  * 0.2
      + time_weight    * 0.1
      + recall_boost   * 0.1
      + current_strength * 0.1
    )

이 스코어링 함수는 "왜 갑자기 오래된 기억이 생생하게 떠오르는가"를 설명한다. consolidation이 높은 기억은 decay가 진행되었더라도 emotion_score와 recall_boost에 의해 검색 순위가 높게 유지될 수 있다.

Section 06

무의식 백그라운드 프로세스의 구현

인간 기억이 정적인 저장 시스템과 근본적으로 다른 이유는 기억이 시간에 따라 변화하기 때문이다. 에빙하우스 망각 곡선, 수면 중 기억 공고화, 감정적 기억 강화는 모두 백그라운드에서 일어나는 능동적 프로세스다. 

def unconscious_consolidation_job():
    """수면에 해당하는 야간 배치 프로세스"""

    # 1. Decay: 오래 회상하지 않은 기억은 희미해짐
    for memory in all_memories():
        days = days_since_recall(memory)
        if days > 7:
            memory.decay_rate *= 0.97  # 에빙하우스 곡선 근사

    # 2. 클러스터링: 의미적으로 가까운 기억들 간 연결 강화
    clusters = semantic_cluster(all_vectors(), eps=0.15)
    for cluster in clusters:
        reinforce_inter_connections(cluster)

    # 3. 재인덱싱: 오늘의 새 경험이 기존 기억의 의미를 바꾼다
    #    ex) "그때 그 사람이 왜 그랬는지 이제 이해가 돼"
    today_context = get_today_memories()
    for memory in find_affected_by(today_context):
        memory.vector = re_encode(
            original  = memory.payload,
            new_context = today_context  # 현재 맥락으로 재해석
        )

    # 4. 감정 가중치 재평가
    #    현재의 감정 상태가 과거 기억의 valence를 미세하게 조정
    for memory in recently_recalled():
        memory.valence = blend(
            memory.valence,
            current_mood.valence,
            alpha=0.05  # 매우 천천히 덮어씀 → 추억 미화의 누적 메커니즘
        )

3번 재인덱싱이 추억 미화의 정확한 구현이다. 기억의 원본 데이터(페이로드)는 변경되지 않지만, 벡터(해석)가 새로운 맥락에 의해 갱신된다. 이는 "사실은 바뀌지 않았는데 기억은 달라진" 심리적 경험의 계산론적 기반이다.

기억은 재생(playback)이 아니라 재구성(reconstruction)이다. 그리고 재구성은 매번 현재의 벡터 공간에서 이루어지기 때문에, 기억은 저장된 순간부터 조금씩 현재를 향해 표류한다.

— 기억 재구성 이론 (Bartlett, 1932; Schacter, 1996)의 계산론적 재해석

Section 08

결론 및 함의

본 논문은 세 가지 핵심 주장을 논증했다.

첫째, 384차원은 인간 장기기억의 압축 차원과 이론적으로 일치하며, 단순 벡터 저장의 관점에서 구현에 충분한 용량이다. 차원 수는 결코 병목이 아니다.

둘째, 진짜 문제는 설계 패러다임에 있다. 벡터DB를 "기억 저장소"로 쓰는 것이 아니라 무의식의 인덱스 레이어로 위치시키고, LLM 컨텍스트를 의식으로, 멀티모달 원본을 재구성 재료로 두는 역할 분리 구조가 필수적이다.

셋째, 기억 왜곡과 추억 미화는 시스템의 버그가 아니라 재인덱싱(재구성) 메커니즘의 구조적 필연이며, 이 특성을 의도적으로 구현함으로써 진정으로 인간적인 기억 시스템을 설계할 수 있다.

인간의 기억은 수십만 차원의 공간 안에서 작동하는 손실 압축 + 근사 검색 시스템이다. 그러나 그것이 압도적으로 강렬하게 느껴지는 이유는 저장된 벡터의 차원 수가 아니라, 저차원 벡터로부터 수십만 차원의 경험을 재구성해내는 디코더— 즉 뇌 전체—의 정교함 때문이다. 우리가 만들어야 할 것은 더 큰 벡터 공간이 아니라 더 나은 디코더다.

— 본 논문의 핵심 명제

향후 과제

본 설계를 실제 AI 에이전트 시스템에 적용하려면 다음 연구가 필요하다: (1) 오감을 단일 임베딩 공간으로 통합하는 범용 감각 인코더 설계, (2) LLM 컨텍스트와 벡터 인덱스 간의 실시간 동기화 프로토콜, (3) 재인덱싱 과정에서 원본 보존과 재해석 갱신 간의 균형 정책, (4) 감정 상태를 실시간으로 추론하여 메타데이터에 반영하는 파이프라인.
